# 配额统计与明细交互优化设计

## 目标

修复使用额度重置卡后菜单栏“已用 Token”仍包含重置前数据的问题，让主窗口和菜单栏的 Token 趋势支持选中日期查看分类明细，并在菜单栏中展示当前账户可用的额度重置卡次数和逐张到期时间。

## 范围

- 使用服务端 `resetsAt - windowDurationMins * 60` 得到精确到秒的额度周期起点。
- 在现有按自然日汇总之外，持久化带事件时间的 Token 正增量，用于精确周期统计。
- 主窗口与菜单栏共用可选择的 Token 趋势图交互规则。
- 解码 `account/rateLimits/read` 已有响应中的 `rateLimitResetCredits`。
- 菜单栏配额区域展示重置卡摘要，并提供逐张到期时间详情页。
- 完成模型、存储、格式化和交互状态的单元测试，并更新 README 与 CHANGELOG。

## 非目标

- 不在 SessionNest 中消费重置卡。
- 不将本地 Token 数量解释为服务端配额的精确计费单位。
- 不增加网络服务、账号系统或遥测。
- 不改变主窗口项目排行和会话表的下钻行为。

## 数据与兼容性

### 精确 Token 增量

`RolloutTokenScanner` 已经从每条 `token_count` 事件计算正增量，但目前立即折叠为自然日。扫描结果需要同时保留：

- 原有 `dailyUsage`，继续写入 `thread_token_daily`，供 v0.1.2 和常规趋势使用。
- 新增 `timedUsage`，按 `threadID + eventAt` 写入 `thread_token_timed`，供精确周期过滤使用。同一会话同一秒出现多条正增量时累加。

新表使用 `CREATE TABLE IF NOT EXISTS` 增量创建，不删除、不重命名、不改变 v0.1.2 已知表或字段。升级后仍持续写旧的每日表。解析器版本上调以为已有数据库重建精确增量；v0.1.2 降级打开时会忽略新表，并按自己的解析器版本重建原缓存，因此数据库保持可打开和可恢复。

### 周期统计

额度周期新增两个边界概念：

- `startTimestamp`：精确到秒，只用于额度周期 Token 过滤。
- 本地自然日：只在已过滤的周期增量需要绘图时用于分组。

模型不再在 SwiftUI `body` 中从每日数组临时推算周期结果。它维护一个发布的周期统计快照；rate-limit 周期边界变化、Token 扫描发布新结果和完整刷新后都重新计算。计算只读取当前周期内的精确增量，随后复用现有项目、会话和 Token 分类汇总语义。

若旧版 App Server 没有完整窗口字段，周期统计保持不可用并回退最近 7 天，不伪造为 0。

## App Server 响应

将 `readRateLimits()` 的返回值扩展为统一的账户用量快照，包含：

- 主 `CodexRateLimitSnapshot`
- `RateLimitResetCreditsSummary?`
- 逐张 `RateLimitResetCredit`

重置卡字段包含 `id`、`resetType`、`status`、`grantedAt`、`expiresAt`、`title` 和 `description`。UI 不显示内部 ID；列表按到期时间升序排列，只把 `status == available` 且未过期的卡计入可用展示。服务端 `availableCount` 与本地过滤结果不一致时，摘要优先使用服务端计数，列表仍忠实展示收到的数据。

额度百分比和重置卡必须从同一响应原子发布。打开菜单栏弹窗时触发一次可合并的轻量账户用量刷新，保留现有十分钟定时刷新和手动完整刷新。

## 图表交互

建立共享的日期选择规则，主窗口和菜单栏都根据指针位置选择最近的 `StatisticsDailyPoint`：

- 悬停临时预览最近日期。
- 单击固定日期；单击空白区域取消固定。
- 选中日期绘制垂直参考线，并在总 Token 与缓存输入两条线上绘制点标记。
- 数据范围变化后，若固定日期不再存在则清除选择。
- 辅助功能提供上一日、下一日和取消选择动作。

详情内容固定为日期、总 Token、输入、缓存输入、输出和推理输出。主窗口在图表下显示完整详情卡；菜单栏在图表下显示紧凑详情，避免浮层被 `NSPopover` 裁剪。

## 重置卡界面

菜单栏“配额”区域在周配额进度条下增加可点击摘要：

- 主文案：`重置卡 可用 N 次`
- 次文案：`最近于 M月d日 HH:mm 到期`
- 字段缺失：`重置卡信息暂不可用`
- 可用数量为 0：`暂无可用重置卡`

点击摘要进入弹窗内的重置卡详情页。详情页提供返回按钮、可用次数、按到期时间排列的卡片以及本地时区的完整到期时间，不提供使用按钮。

## 错误与降级

- 轻量刷新失败时保留上一次成功快照，不把次数改成 0。
- 精确 Token 表尚在重建时显示扫描进度，完成后自动替换周期快照。
- 无精确周期数据时继续使用既有最近 7 天回退标题。
- 单点趋势仍可选择，并明确绘制点标记。

## 验收

- 同一自然日中重置前后都有 Token，周期统计只包含服务端周期起点之后的正增量。
- App 跨重置时间关闭后再打开，能从本地 rollout 重建正确结果。
- v0.1.2 数据库可直接升级；新增表后 v0.1.2 仍能打开数据库并重建它理解的缓存。
- 重置卡摘要、列表、排序、到期格式、空状态和字段缺失状态正确。
- 主窗口和菜单栏均支持悬停、固定、取消及完整 Token 分类详情。
- 完整 `bash Scripts/check.sh` 通过。
