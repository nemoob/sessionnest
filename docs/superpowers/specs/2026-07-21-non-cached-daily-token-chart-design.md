# 每日非缓存 Token 图表设计

## 背景

状态栏弹框当前把 Codex 日志中的 `total_tokens` 直接展示为“本周期每日 Token”。
`total_tokens` 包含缓存输入，缓存占比较高时会出现 `105 亿` 这类容易被误解为额度消耗的数值。
服务端周额度百分比仍是唯一权威的额度数据，本地图表只用于观察本机可读取会话的 Token 变化。

## 方案选择

考虑过三种方案：

1. 继续展示总 Token，只加强说明。改动最小，但巨大缓存值仍会主导柱状图。
2. 同时绘制总 Token 和非缓存 Token。信息完整，但 7 天紧凑图表会变得拥挤。
3. 默认绘制非缓存 Token，选中某天后补充总 Token、缓存输入和占比。

采用方案 3。它保留完整信息，同时让默认柱状图更接近用户理解的“实际新增消耗”。

## 数据口径

非缓存 Token 按以下公式计算：

```text
nonCachedTokens = max(0, totalTokens - cachedInputTokens)
```

缓存输入已经包含在输入 Token 和总 Token 中，因此不再额外加减输出或推理 Token。
当异常日志出现 `cachedInputTokens > totalTokens` 时结果按 0 处理，避免负数进入图表。

缓存占比按 `cachedInputTokens / totalTokens` 计算，并限制在 `0...100%`；总 Token 为 0 时显示 0%。

## 界面变化

- 当前周期标题改为“本周期每日非缓存 Token”。
- 无服务端周期信息时，回退标题改为“最近 7 天每日非缓存 Token”。
- 每根柱条顶部数字、柱高和无障碍描述都使用非缓存 Token。
- 点击有记录的日期后，详情区展示：
  - 当日非缓存 Token；
  - 当日总 Token；
  - 缓存输入 Token 与缓存占比。
- 说明文字明确该图表按本机日志估算，且不代表服务端额度消耗。
- 周额度剩余比例、重置时间和刷新逻辑保持不变。

## 实现边界

只修改展示层的派生计算和文案：

- 不修改日志扫描算法；
- 不修改 SQLite 表结构或历史数据；
- 不增加轮询、后台任务或能耗；
- 不修改主窗口现有的总 Token 统计口径。

## 测试

- 验证非缓存 Token 的正常计算和负数保护。
- 验证缓存占比的正常计算、零值和上限保护。
- 验证图表标题与说明文字使用“非缓存”口径。
- 验证柱高按非缓存 Token 缩放。
- 运行项目全量格式、单元测试和应用打包检查。

## 验收标准

给定某天总 Token 为 `10,500,000,000`、缓存输入为 `9,900,000,000`：

- 柱条显示 `6 亿`，而不是 `105 亿`；
- 选中后显示总 Token `105 亿`、缓存输入 `99 亿` 和缓存占比；
- 周额度条仍显示 Codex 服务端返回的剩余比例。
